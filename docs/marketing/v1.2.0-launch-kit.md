## 定位

BlueTeam Log Analyzer 是面向蓝队应急响应、HVV/重保和值守复盘的离线日志分析器。它支持 Windows、Linux、Web access、P0 结构化安全日志，输出终端报告、HTML、JSON、CSV、IOC 和 SARIF。

v1.2.0 是“可扩展内核 / 博物馆地基版”：Parser Registry、Detector Registry、`parse_content()` 和 `--type` 已经就位，后续 Remote Collector、Host Triage、P0 adapter split 和离线 IR Workbench 可以按展厅继续扩建。

### 正文

BlueTeam Log Analyzer v1.2.0 is the extensible-core release.

This release does not chase another pile of rules. It lays down the architecture BLA needs to grow into a larger offline incident-response workbench.

Highlights:

- Parser Registry: add new log sources without growing a central router.
- Detector Registry: compose new detection logic without stuffing everything into one engine file.
- `parse_content()`: reuse the same pipeline for future collectors, UI inputs, and in-memory logs.
- CLI `--type`: force `web-access`, `linux-auth`, `p0-security`, `windows-xml`, or `generic` when auto-detection is ambiguous.
- Pipeline cleanup: enrichment now happens once in the main flow.
- Architecture docs: the long-term museum foundation is documented.

## FAQ

### 它是不是 AI 工具？

不是。BLA 是离线、可审计、规则和管线驱动的日志分析工具。

### 它能替代 SIEM 吗？

不能，也不应该。BLA 更像离线应急、日志复盘和证据打包工具，可以补充 SIEM 工作流。

### 为什么是 Python？

Python 对日志解析、应急脚本、报告生成和扩展 API 很合适。v1.2.0 的注册表结构也给未来可选的 native backend 留了位置。
