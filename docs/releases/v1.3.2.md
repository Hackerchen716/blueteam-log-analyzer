# BlueTeam Log Analyzer v1.3.2

v1.3.2 是一次可信度与发布卫生更新：收紧解析失败处理、登录类检测时间窗口、RDP 专项过滤、终端输出清洗，并同步修复 README、打包清单和发布流程。

## 修复与增强

- P0 结构化日志支持 pretty JSON object、JSON array、JSONL、CSV 和 key=value；非法 JSON 会记录 `parse_errors`，不会被当作“正常空结果”吞掉。
- VPN 认证日志只有明确成功/失败时才标记 `successful-login` / `failed-login`；MFA challenge 等中间状态会作为认证观察事件处理。
- 防火墙敏感端口只有明确 `allow/accept/pass/permit/允许/放行` 时才报“放行”；动作缺失时降为“动作未知”，避免误报暴露服务。
- Windows Kerberos 4771 预认证失败加入失败登录标签，可进入暴力破解/密码喷洒聚合。
- 暴力破解、密码喷洒和 `cn-hvv` 爆破后成功登录增加时间窗口与顺序校验，避免跨天/月低频失败被误聚合。
- Web 攻击同源 IP 仍可同时产生流量异常事件，避免攻击 + 高频请求组合被隐藏。
- Incident 关联会合并重叠事件/告警组，减少近重复案件。
- 终端报告清理日志中的 OSC/CSI/control 字符，防止原始日志污染交互终端。
- Remote Workspace 的 `--exit-on` 与主 CLI 保持一致，并增加 `--max-alerts` / `--syslog-year` 参数校验。
- SARIF 对远程来源路径保留 `original_source_file`，同时生成更稳定的 artifact URI。

## 文档与发布

- README 更新为 v1.3.2 当前能力说明，移除旧版本堆叠和过时示例。
- 新增 `MANIFEST.in`，让 sdist 包含 docs、sample logs 和 P0 fixtures，便于源码包复测。
- 发布 checklist 和 publish workflow 增加版本/tag 校验、pytest、规则校验、Remote Workspace help、wheel 安装烟测和 sdist 内容检查。

## 边界

- `--rdp` 是专项模式，只保留 `LogonType=10` 且带远程来源 IP 的 Windows 4624/4625 事件；通用 Windows 解析仍保留完整事件。
- 时间窗口聚合会降低长期低频失败的误报；如果需要适配特殊场景，可通过 `--config` 或 `BLA_THRESHOLD_*` 覆盖阈值。
