# BlueTeam Log Analyzer v1.4.3

v1.4.3 是一次面向发布交付的稳定性和检测覆盖增强，重点提升真实 P0/HVV 日志兼容、大日志报告控制、输出安全和 CLI 打包边界。

## 主要变化

- CLI 主入口迁移到 `bla/cli/main.py`，`bla_cli.py` 保留为向后兼容 shim；包入口改为 `bla.cli.main:main`，发布包会校验新入口文件。
- JSON 报告新增 `--json-events-limit`、`--no-json-events` 和 `--raw-line-limit`，用于控制大日志场景下 `report.json` 的事件数量和原始行长度。
- 终端、CLI、Remote Workspace、bundle、HTML、JSON、CSV、SARIF、manifest 等输出路径和错误展示统一使用报告清洗逻辑，降低控制字符和常见 secret 泄漏风险。
- Shell History 新增数据外传命令检测，覆盖 `scp`、`rsync`、`curl --upload-file`、`wget -T`、`nc < file` 以及压缩命令管道到 `curl/nc` 等行为，并生成 `EXFIL-001` 告警。
- P0/HVV parser 扩展真实厂商字段兼容：
  - WAF: `request_uri`、`request_url`、`full_url`、`attack_name`、`signature_name`、`src_addr`。
  - DNS: `dns_query`、`rrname`、`threat_category`、`domain_category`、`query_category`、`response_ip`。
  - Proxy/SWG: `request_url`、`request_uri`、`full_url`、`url_category`、`threat_category`、`policy_action`。
  - Firewall/NAT: `policy_action`、`event_action`、`rule_action`、`session_action`。
  - VPN/Application: `auth_result`、`login_result`、`auth_status`、`login_status`、`service_name`、`app_name`。
  - Bastion: `operation_type=file_upload/file_download`、`target_asset`、`asset_name`、`file_name`。
  - EDR/XDR: `detection_name`、`process_name`、`command_line`、`technique_id`、`mitre_id`。
- P0 外传流量归一化增加方向约束：明确 `inbound/ingress/download/server-to-client` 不作为外传；`cs-bytes`、`request_body_bytes`、`orig_bytes` 等只在语义明确时参与 `P0-EXFIL-001`。
- 凭据访问检测改为 selector 预筛，新增 Shell 数据外传检测也通过 DetectorRegistry 注册，保持 detector 边界清晰。
- CI 测试安装 dev extra 并运行 coverage gate；release check 会检查 `bla/cli/main.py` 是否进入 wheel/sdist，并执行 v1.4.3 冒烟场景。

## 使用方式

大日志交付时可以限制 JSON 事件明细：

```bash
bla logs/ --out report/ --json-events-limit 50000 --raw-line-limit 500
bla logs/ --out report/ --no-json-events
```

P0/HVV 结构化日志仍可使用自动识别，也可以显式指定：

```bash
bla p0-security.log --type p0-security --profile cn-hvv --out report/ --exit-on none
```

## 安全边界

- 不引入默认联网行为。
- HTML 报告继续离线可用，不依赖 CDN。
- Remote Workspace 和 `remote-log` 继续保持只读、限量、超时、可审计。
- Parser 只把日志转为 `LogEvent`，Detector 只生成 `DetectionAlert`，Correlation 保留原始事件语义。
