# BlueTeam Log Analyzer v1.4.4

v1.4.4 是一次面向真实样本验证和发布可靠性的稳定性增强，重点提升 Windows 攻击日志兼容、EDR 导出日志兼容、C2/外传/防御规避/UAC 提权检测、输出脱敏、manifest provenance 和安装后发布包 smoke。

## 主要变化

- 新增 `windows-json` parser，并通过 ParserRegistry 注册；支持 Winlogbeat/Elastic 嵌套字段、OTRF/Mordor JSONL、JSON object、JSON 数组和多对象 pretty JSON，不扩大中央 `if/elif` 分支。
- 新增 `edr-xlsx` parser，并通过 ParserRegistry 注册；使用标准库 ZIP/XML 流式读取 EDR/XDR `.xlsx` 进程事件导出，支持显式 `--type edr-xlsx` 和复制成 TSV/CSV 后的 `parse_content`。
- Windows XML/EVTX 解析增强命名空间、单引号 XML、Sysmon/安全日志字段归一，提升对真实 Windows 安全日志、Sysmon DNS、注册表、进程和网络事件的兼容性。
- Windows 检测增强 C2 外联、DNS 隧道/外传线索、auditpol 防御规避、UAC 注册表劫持、可疑执行、横向移动和凭据访问覆盖，并收紧 LDAP 等高噪声误报。
- EDR Excel 检测增强用户目录无签名伪装进程、临时目录投放、随机名可执行文件、WebRoot 可执行文件、随机计划任务删除、随机目录 ACL 修改和 `netsh interface portproxy reset` 线索；安全组件 DLL 加载降为上下文噪声，不再单独作为高危核心证据。
- 终端报告新增 EDR 进程链树状展示，并把 EDR 时间线拆成多行字段，避免长路径和命令行挤在同一行。
- P0/HVV 结构化日志继续扩展 WAF、DNS、Proxy/SWG、Firewall、VPN、EDR、堡垒机和应用审计字段别名，保持 selector 预筛和 detector 边界。
- Linux auth/secure、Apache/Nginx、Shell History、P0 CSV/JSON/JSONL/key=value 和通用文本日志的恶意文本、secret、terminal control、HTML escaping 回归覆盖继续扩展。
- 普通 CLI `--out` manifest 复用 pipeline 输入 provenance，能够区分不同目录下的同名日志，记录输入/输出 sha256，同时避免暴露本地绝对路径。
- `manifest.options` 对 `--config`、`--allowlist`、`--rules`、`--geoip-cache` 只保留清洗后的安全显示名，避免泄漏操作员本地目录或明显 secret。
- 输出链路继续清洗终端、HTML、JSON、CSV、IOC、SARIF 和 manifest 中的不可信文本；SARIF 保持 2.1.0；HTML 报告继续离线可用。
- `scripts/release_check.py --build` 增加安装后 wheel smoke：离线安装 wheel 后执行 `bla --version`、Python API、content/file 分析、CLI JSON、完整 bundle、同名输入 manifest 和 manifest options 脱敏校验。

## 使用方式

Windows JSON 事件日志可以自动识别，也可以显式指定：

```bash
bla windows-events.jsonl --type windows-json --out report/ --exit-on none
```

EDR/XDR Excel 导出可以自动识别，也可以显式指定：

```bash
bla edr-process-export.xlsx --type edr-xlsx --out report/ --exit-on none
```

生成完整交付包时，manifest 会记录输入和输出哈希，并用相对标签区分同名文件：

```bash
bla logs/ --out report/ --exit-on none
```

## 安全边界

- 不引入默认联网分析，不上传日志。
- HTML 报告继续离线可用，不依赖 CDN。
- Remote Workspace、`ssh` 和 `remote-log` 继续保持只读、限量、超时、可审计，不扩展任意命令执行。
- Parser 只生成 `LogEvent`；Detector 只生成 `DetectionAlert`；Correlation 保留原始事件；Output 只生成 HTML/JSON/CSV/IOC/SARIF/manifest。
- 发布包 smoke 使用本地构建产物和 `--no-index` 离线安装，不访问外部包源。
