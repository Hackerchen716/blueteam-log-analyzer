# BlueTeam Log Analyzer v1.3.7

v1.3.7 聚焦终端报告可读性、攻击路径表达和产品归属展示，不改变现有解析、检测、远程采集和报告输出用法。

## 修复与增强

- CLI 终端报告横幅新增“南京澄安科技有限公司出品”，并保留版本、离线运行和 No AI 属性。
- ATT&CK 攻击链从阶段清单调整为阶段拓扑，按应急 kill chain 顺序展示命中阶段、事件数量和技术编号。
- Incident 案件视图新增攻击拓扑，按“源 IP / 资产 / 账号 -> 攻击阶段 -> 关键事件分支”还原路径。
- P0/HVV 多源案件可在终端中直接看到 WAF、应用、EDR、DNS、代理、防火墙之间的分支关系。
- Linux auth.log 这类单源爆破场景会以“源 IP / 资产 / 账号 -> 身份突破 / 持久化 / 权限提升”展示账号扇出和后续行为。
- 新增 `docs/demo-cases.md`，整理真实样本、候选公开数据源、覆盖矩阵、复现命令和发布前不足。

## 验证重点

- 横幅中文宽度已按 East Asian Width 计算，避免中文公司名导致边框错位。
- 拓扑展示只修改终端输出层，不改变事件、告警、Incident、JSON、CSV、SARIF 或 manifest 的结构化数据。
- 发布前继续运行 `python3 scripts/release_check.py --build`、`python3 -m pytest -q`、`python3 -m unittest discover -s tests -v`、`python3 -m compileall -q bla bla_cli.py setup.py tests` 和 `python3 bla_cli.py validate-rules --strict-metadata`。
