# BlueTeam Log Analyzer v1.2.2

v1.2.2 聚焦 Windows 应急案件还原能力，增强账号、特权组、远程登录和来源实体之间的自动关联。

## 重点更新

- 新增 `WIN-CHAIN-001`：通用关联新建账户、启用/改密、加入特权组后 10 分钟内发生网络/RDP 成功登录的链路。
- Incident 聚合优先围绕核心账号、操作者、来源 IP、来源工作站和资产组织，减少“未知来源/其他”。
- 终端报告新增 incident 级“攻击路径还原”，按时间串联关键事件。
- 普通 Windows `4688` 进程创建不再默认映射 `T1059`；只有高危命令或 LOLBins 命中时才进入执行阶段。
- Top IP 输出过滤 `-`、loopback、localhost 等本机/空来源，并单独展示本机/空来源统计。
- 终端与 HTML 中的时区感知时间默认显示为 UTC+8，同时保留原始 UTC。
- 轻量 YAML 解析器不再通过 Python 字面量解析正则字符串，避免 `invalid escape sequence` 警告。

## 兼容性

- Python 版本要求不变：3.9+。
- `python-evtx` 仍为可选依赖；分析二进制 `.evtx` 时需要安装 `blueteam-log-analyzer[evtx]` 或先用 `wevtutil` 导出 XML。
- JSON 事件和告警字段保持兼容，`ParseStats` 增加 `top_local_ips` 用于区分有效远程源与本机/空来源。

## 验证

- `python3 -m compileall -q bla bla_cli.py setup.py tests`
