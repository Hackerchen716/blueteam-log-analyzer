# BlueTeam Log Analyzer v1.4.2

v1.4.2 聚焦输出安全、解析稳定性和 Shell History 现场语义，让交付包在真实日志和不可信输入下更稳。

## 主要变化

- IOC 文本导出复用统一报告清洗逻辑，去除终端控制序列，并脱敏常见 token、Bearer、password、cookie 等敏感字段。
- 终端报告、CLI 和远程采集工作区展示不可信文本时，会清洗文件名、异常字符串和证据摘要，避免恶意控制字符或常见 secret 污染输出。
- 通用 fallback 解析器改为从文件路径逐行读取，不再为了 fallback 解析一次性读完整文件；超出行数上限时仍保留截断提示。
- Shell History 解析会从 `server:/home/user/.zsh_history`、`/home/user/.bash_history`、`/Users/user/...` 等来源提取账号和资产上下文，并保留 zsh 扩展时间戳。
- Shell History 生成 Incident 时会把提取到的账号/资产写入标题、描述和证据，例如 `alice 的 Shell 凭据访问轨迹`。
- GeoIP 地图区块继续只渲染可定位公网源 IP，同时在报告脚注中说明已排除的内网/回环/保留源 IP 数量，以及缺少地理数据的公网源 IP 数量。
- Release gate 新增 v1.4.2 冒烟场景，覆盖 Shell History 账号上下文、Geo 计数说明和 IOC 脱敏。

## 使用方式

```bash
bla logs/ --out report/ --geoip-cache geo-cache.json --exit-on none
```

目录扫描仍会保留 `.bash_history`、`.zsh_history` 这类主机证据文件。若文件路径包含账号或远程主机上下文，BLA 会尽量把这些上下文带入 Incident 和报告证据。
