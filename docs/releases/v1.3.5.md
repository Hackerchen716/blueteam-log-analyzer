# BlueTeam Log Analyzer v1.3.5

v1.3.5 聚焦输入边界和远程取证可靠性，不改变主要 CLI 用法。

## 修复与增强

- 本地目录扫描会跳过隐藏目录，避免 `.git`、`.cache` 等内部文件被默认纳入日志分析范围。
- 从目录输入收集文件时，符号链接必须解析在该输入根目录之内；指向根目录外部的链接文件会被跳过。
- Glob 输入现在会对匹配到的目录继续执行同一套目录收集规则，而不是把目录路径直接交给解析层。
- Remote Workspace 的 `fetch_file()` 在远程文件大小预检之后，会在实际传输命令上追加 `head -c max+1` 硬上限。
- 如果远程文件在预检和实际拉取之间变大，BLA 会删除超过上限的临时副本并返回明确错误。

## 验证重点

- 本地输入目录中的隐藏目录和越界符号链接不会进入分析文件列表。
- 远程文件拉取同时覆盖 `wc -c` 预检和实际传输上限，避免竞态导致超大文件落地。
- 发布前继续运行 `python3 scripts/release_check.py --build`、`python3 -m pytest -q`、`python3 -m unittest discover -s tests -v`、`python3 -m compileall -q bla bla_cli.py setup.py tests`、`python3 bla_cli.py validate-rules --strict-metadata` 和 `python3 bla_cli.py ssh --help`。
