# BlueTeam Log Analyzer v1.3.4

v1.3.4 聚焦大日志性能和 P0 解析结构治理，不改变主要 CLI 用法。

## 修复与增强

- Windows XML 文件入口改为按 `<Event>...</Event>` 块流式解析，避免 `auto_parse()` 对大型 XML 导出整文件读入。
- P0 CSV 文件入口改为逐行解析；P0 JSON object / JSON array / JSONL 文件入口改为增量解码，JSON 数组不再先整体 materialize 为 Python list。
- P0 结构化日志增加 adapter 注册表雏形，WAF、VPN、EDR、DNS、代理、防火墙、堡垒机和应用日志按 adapter 元数据路由，减少主解析器继续膨胀。
- Detector Registry 增加候选事件 selector，默认检测器先按 tag、event_id 或 category 预筛，再进入各自聚合逻辑，降低多轮全量扫描压力。
- Windows XML 文件末尾存在未闭合 `<Event>` 时会计入 `parse_errors`，避免截断日志被误认为完整解析成功。
- P0 JSON 文件遇到截断或坏记录时保留已解析事件，并在 `parse_errors` 暴露解码错误。

## 性能基准

在本地 `python3 bla_cli.py benchmark --size-mb 5` 的合成 P0 JSONL 样本上，本次验证约为：

- `5.00 MB`
- `34,165` events
- `44` alerts
- `4` incidents
- about `1.9s` elapsed
- about `2.6 MB/s` throughput

## 验证重点

- Windows XML、P0 CSV、P0 JSON array、P0 JSON wrapper 和 JSONL 回归样本。
- 默认 detector selector 不应改变现有告警、incident 和时间线结果。
- 发布前继续运行 `python3 scripts/release_check.py --build`、`python3 -m pytest -q`、`python3 -m unittest discover -s tests -v`、`python3 -m compileall -q bla bla_cli.py setup.py tests` 和 benchmark。
