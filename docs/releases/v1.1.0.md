# BlueTeam Log Analyzer v1.1.0

## Highlights

- Adds a reusable analysis pipeline for CLI, collectors, tests, and future UI layers.
- Centralizes the package version in `bla/__version__.py`.
- Adds release checklist documentation, a GitHub release checklist issue template, and a PyPI Trusted Publishing workflow.
- Hardens allowlist suppression, custom rule loading, alert correlation, SARIF attribution, and concurrent event ID generation.

## Security And Correctness Fixes

- Empty `maintenance_windows` or `suppressions` allowlist objects no longer suppress every event.
- Windows 4624 success events now participate in `cn-hvv` "爆破后成功登录" detection.
- Event IDs are generated under a lock for parallel parsing.
- Invalid custom YAML rules fail fast during rule directory setup.
- Near-duplicate incident groups are suppressed when they describe the same entity and event set.
- SARIF alert locations keep the first matching source file instead of being overwritten by later files.

## Validation

- `python3 -m unittest discover -s tests -v`
- `python3 bla_cli.py validate-rules --strict-metadata`
- `python3 bla_cli.py sample_logs/auth.log --exit-on none --no-color`
- `python3 bla_cli.py sample_logs/access.log --exit-on none --no-color`
