# BlueTeam Log Analyzer v1.3.0

v1.3.0 引入 Remote Workspace，让分析机通过 SSH 进入远程日志目录，并把远程日志拉回本地完成分析。目标机不需要安装 Python、pip 或 BLA。

## 新增

- 新增 `bla ssh TARGET` 交互入口，支持 OpenSSH 主机别名、端口、私钥和初始目录。
- 工作台内支持 `ls`、`cd`、`pwd`、`find`、`tail`、`bla FILE` 和 `bla journalctl:UNIT`。
- `bla FILE` 会通过 SSH 读取远程文件，在本机复用现有 `parse -> enrich -> detect -> correlate -> report` 流程。
- 报告中的文件来源标记为 `host:/remote/path` 或 `host:journalctl:unit`，便于交接证据。

## 边界

- 远程侧只执行白名单只读命令，不执行自动修复。
- 不要求目标机安装 Python、pip、BLA 或第三方依赖。
- 当前版本聚焦 Linux/POSIX SSH 工作台；Windows WinRM/PowerShell 工作台和完整 collector 证据包留到后续版本。

## 示例

```bash
bla ssh root@192.168.1.20
cd /var/log/nginx
bla access.log --out case-nginx --exit-on none
bla journalctl:ssh --out case-ssh --exit-on none
```
