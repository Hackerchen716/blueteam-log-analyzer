# BlueTeam Log Analyzer v1.3.3

v1.3.3 是一次产品打磨、性能和发布质量更新：不改变主要使用方式，重点优化 P0 结构化日志热路径、报告可读性、远程工作台边界和发布前可重复验证。

## 修复与增强

- P0 结构化安全日志优先使用 `log_type` / `source_type` / `type` 等显式字段分流到 WAF、VPN、DNS、代理、防火墙、EDR 和应用日志处理逻辑，减少每行反复做大范围正则判断。
- `compute_stats()` 不再为计算起止时间排序全量时间戳，改为单次遍历维护最早/最晚时间，降低大日志 CPU 和内存开销。
- RFC1918 内网地址判断增加缓存，减少富化、置信度调整和报告统计中的重复 IP 解析。
- enrichment 里的 source type、资产角色和域名类型判断复用预编译正则，减少大批量事件归一化开销。
- 凭据访问检测减少对完整 `details` 的重复字符串拼接，只检查高价值字段和已有标签，降低检测热路径成本。
- JSON 报告新增顶层 `summary`，固定输出风险、文件数、事件数、告警数、案件数、时间线数量、建议数量和级别分布，便于 API、工单和二次处理直接读取。
- `--out` 标准报告目录新增 `manifest.json`，记录输入哈希、输出哈希、工具版本、报告限制、解析异常和压制事件数，便于交付复核与自动化归档。
- 分享型报告会清理终端控制序列，并遮蔽常见 `token`、`cookie`、`password`、`Authorization: Bearer ...` 等敏感字段。
- JSON/HTML 时间线和 CSV 原始日志字段增加截断元数据或可见提示，避免使用者误以为报告中展示的是全量字段。
- Remote Workspace 拒绝以 `-` 开头的 SSH 目标并在 OpenSSH 参数中插入 `--`，避免目标名被解释为本地 ssh 参数；远程文件拉取和 `journalctl:` 输出增加大小与超时上限。
- `bla explain --format markdown` 会对报告字段做 Markdown/HTML 转义并清理控制序列，降低把外部日志直接粘贴进工单时的渲染风险。
- `validate-rules` 对外置规则增加常见 ReDoS 形态提示，帮助维护者在发布前发现容易拖慢大日志分析的正则。

## 性能基准

在本地 `python3 bla_cli.py benchmark --size-mb 5` 的合成 P0 JSONL 样本上，优化后约为：

- `5.00 MB`
- `34,165` events
- `44` alerts
- `4` incidents
- about `1.8s` elapsed
- about `2.7 MB/s` throughput

该基准用于观察相对变化，不代表所有真实日志格式的绝对吞吐。
