# BlueTeam Log Analyzer v1.3.6

v1.3.6 聚焦 Remote Collector 收口、远程采集审计和 GitHub Actions 运行时预适配，不改变现有本地分析用法。

## 修复与增强

- 新增 `bla remote-log TARGET PATH` 脚本化入口，可通过 SSH 只读采集远程日志并在本机复用现有分析流程。
- `remote-log` 支持 `--tail N` 和可重复 `--grep TEXT`，用于只采集远程文件子集，减少大日志完整拉取成本。
- `remote-log` 支持 `journalctl:UNIT`、`--out`、`--json`、`--html`、`--csv`、`--ioc`、`--sarif`、`--profile`、`--type`、`--exit-on`、`--rdp` 等现有分析参数。
- Remote Workspace 中的 `bla FILE` 同步支持 `--tail`、`--grep` 和 `--audit-json`。
- 标准报告 `manifest.json` 增加 `remote_collection`，记录远程来源、采集方式、tail/grep 参数、采集大小和哈希。
- 新增 `--audit-json FILE`，可单独写出远程采集审计记录，便于工单、复盘或证据链交接。
- GitHub Actions 增加 `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true`，提前适配 Node.js 24 action runtime。
- release gate 和 CI smoke 增加 `bla remote-log --help`，确保 Remote Collector CLI 在发布前可用。

## 验证重点

- `remote-log --help`、`ssh --help`、规则校验、benchmark、样例烟测和 P0 报告包均纳入 `scripts/release_check.py`。
- 新增回归测试覆盖远程 `tail + grep` 命令构造、manifest `remote_collection` 和 `--audit-json` 输出。
- 发布前继续运行 `python3 scripts/release_check.py --build`、`python3 -m pytest -q`、`python3 -m unittest discover -s tests -v`、`python3 -m compileall -q bla bla_cli.py setup.py tests` 和 `python3 bla_cli.py validate-rules --strict-metadata`。
