# BlueTeam Log Analyzer v1.4.0

v1.4.0 聚焦攻击链和应急案件视图的一致性。这个版本把 ATT&CK 阶段拓扑、Incident 攻击拓扑、告警阶段和报告展示收敛到同一套事件语义上，让多源日志分析结果更适合值守研判、案件复盘和工单交付。

## 关键变化

- 攻击链按唯一事件计数，告警只用于补充阶段和技术信息，避免 alert + event 双重计数。
- ATT&CK 攻击链、Incident 攻击拓扑和告警详情统一使用事件语义归类，减少不同视图之间的阶段口径差。
- P0/HVV 多源样本可稳定还原 `初始访问 -> 主机失陷 -> 命令控制 -> 数据外传` 链路。
- `T1505.003` / Webshell 归入 `主机失陷`，不再被误挂到命令控制或执行阶段。
- `T1041` 数据外传进入顶部 ATT&CK 攻击链展示，不再只出现在 Incident 视图。
- 暴力破解和密码喷洒归入 `身份突破`，sudo shell 归入 `权限提升`，敏感文件探测归入 `侦察`。
- 终端和 HTML kill chain 阶段列表同步扩展，保持不同报告形态的展示一致。

## 验证重点

- 新增攻击链事件去重计数回归测试。
- 新增告警阶段与事件阶段一致性回归测试。
- 新增 Web 侦察、sudo shell、P0 EDR/XDR、P0 golden incident 相关回归测试。
- 发布检查继续覆盖 compileall、pytest、unittest、规则元数据校验、Remote Workspace help、benchmark、样例 smoke 和报告包生成。

## 兼容性说明

- 本次调整不引入联网依赖，不改变离线运行模式。
- 标准报告包文件仍为 `index.html`、`report.json`、`events.csv`、`iocs.txt`、`report.sarif` 和 `manifest.json`。
- 攻击链和告警阶段语义会比旧版本更精确；如果下游系统依赖旧的阶段文本，请在升级前确认映射关系。
