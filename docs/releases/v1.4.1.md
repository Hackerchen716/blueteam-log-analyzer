# BlueTeam Log Analyzer v1.4.1

v1.4.1 聚焦报告可读性和离线地理分布，让大批量真实日志更适合值守研判、复盘和交付。

## 主要变化

- HTML 报告新增可选攻击源地理分布：只读取日志自带地理字段或 `--geoip-cache` 本地 JSON 缓存，不联网查询 IP。
- GeoIP 缓存路径不可读或 JSON 损坏时不会中断报告生成；报告会继续使用日志内地理字段，仍无结果则隐藏地图。
- 没有可定位公网源 IP 时，HTML 报告不展示地图区块，避免默认点位或误导性可视化。
- 暴力破解、密码喷洒等高重复告警在 HTML 展示层合并，`report.json`、`events.csv` 和 SARIF 保留原始明细。
- 仅源 IP 不同的身份突破案件在 HTML 展示层合并成活动视图，突出 Top 来源 IP、账号、资产和处置动作。
- 顶部严重/高危/中危统计卡支持点击跳转到对应告警或时间线，并自动应用过滤条件。
- 报告所需地图资产打包在 Python 包内，HTML 生成过程不依赖 CDN 或在线地图服务。
- 新增 Bash/Zsh shell history 解析与命令轨迹检测，覆盖主机枚举、远程工具下载、TTY 升级、SUID/sudo 提权、Linux 凭据文件读取和清痕命令；目录扫描会保留 `.bash_history`、`.zsh_history` 这类主机证据文件，同时继续跳过 `.git` 等隐藏目录。
- Shell history 单源案件会使用 `Shell 凭据访问轨迹`、`Shell 清痕轨迹` 等可研判主体，不再在报告中显示 `未知实体` 或 `未知来源`。

## 使用方式

```bash
bla logs/ --out report/ --geoip-cache geo-cache.json --exit-on none
```

GeoIP 缓存示例：

```json
{
  "8.8.8.8": {
    "country": "United States",
    "regionName": "California",
    "city": "Mountain View"
  }
}
```

也可以使用日志事件中已有的 `country`、`geo_country`、`src_country`、`city`、`regionName` 等字段。内网、回环、保留地址和无法匹配国家/地区的 IP 不会进入地图。
