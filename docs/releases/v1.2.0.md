# BlueTeam Log Analyzer v1.2.0

v1.2.0 是“可扩展内核 / 博物馆地基版”。这一版不追求继续堆更多规则，而是把后续无限扩建需要的主结构先立起来：解析器注册表、检测器注册表、内存日志解析入口、强制日志类型选择，以及一份可持续迭代架构说明。

## Highlights

- Adds `ParserRegistry` so new log sources can be registered instead of added to a central routing chain.
- Adds `DetectorRegistry` so new detectors can be composed by profile without growing one monolithic engine.
- Adds `parse_content()` for Remote Collector and future in-memory log sources.
- Adds CLI `--type` to force a parser when automatic detection is ambiguous.
- Avoids duplicate enrichment between pipeline and detection.
- Documents the long-term extensible-core architecture in `docs/architecture.md`.

## Compatibility

- Existing `bla <path>` usage remains supported.
- Existing `validate-rules`, `benchmark`, and `explain` subcommands remain supported.
- Existing JSON/HTML/CSV/IOC/SARIF report formats remain compatible.
